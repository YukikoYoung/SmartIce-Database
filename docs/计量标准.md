# 供应链计量单位 & 价格体系标准（含逻辑与概念说明 · 精简合并版）

## 0. 文档说明  
本标准用于：  
- 连锁餐饮供应链管理  
- 采购、仓储、配送  
- 成本核算（COGS）  
- 配方管理（Recipe）  
- ERP/中台系统数据字段统一  
- 未来 AI 自动化采购/库存管理  

设计原则：**简单、统一、不易混淆、可扩展、可自动换算。**

---

# 一、核心概念与标准逻辑

供应链的计量体系必须遵循以下逻辑链：

```
包装单位（供应商交付）  
   → 基础单位（系统成本核算）  
      → 使用单位（厨房配方/出品）
```

### 为什么这样设计？  
- **包装单位**：用于“采购与收货”，不能直接用于库存扣减或成本。  
- **基础单位**：是所有库存在系统中的最“小单位”，成本、库存均以此来记录。  
- **使用单位**：厨房制作菜品的操作单位，必须可被换算成基础单位。  

**三者分别对应三类价格：包装价格、基础价格、使用价格。**  
这是保证成本准确、库存准确、配方不混乱的根本。

---

# 二、基础单位 Base Unit（系统成本与库存核心单位）

### 2.1 定义说明  
Base Unit 是供应链系统的最小计量单位，用于：  
- 成本核算  
- 库存数量  
- 出入库换算  
- 配方消耗折算  

所有物料必须最终“归一到基础单位”，否则成本无法精确管理。

---

### 2.2 基础单位（合并精简后的版本）

| 分类 | 中文 | 英文 | 说明 |
|------|--------|--------|--------|
| 重量 | 克 | g | 成本核算的通用基础单位 |
| 重量 | 公斤 | kg | 显示用，系统可自动换算为 g |
| 体积 | 毫升 | ml | 液体通用基础单位 |
| 体积 | 升 | L | 显示用，自动换算 ml |
| 计数 | 个 | piece | 所有粒/根/支/片统一为 piece |
| 小包装 | 包 | pack | 最常见小包装单位 |

### 2.3 设计逻辑  
- 计数类单位统一为 **piece** → 避免 乱用 “粒/根/支/片”等导致系统难维护  
- 重量统一使用 **g / ml** → 便于自动换算  
- 小包装统一为 **pack** → 便于计算“件 → 包 → 克”链路  

---

# 三、包装单位 Package Unit（采购单位）

### 3.1 定义说明  
Package Unit 由供应商提供，是采购时签约、收货的单位。

⚠ **不能直接用于库存扣减，也不能直接用于成本核算。**

原因：  
- 包装单位不标准（1箱可能装10包，可能装12包）  
- 不同供应商包装不同  
- 必须转换为 Base Unit 才能管理库存

---

### 3.2 包装单位（精简合并后）

| 中文 | 英文 | 说明 |
|------|--------|--------|
| 件 | case | 一级外包装 |
| 箱 | box | 外箱/中箱统一为 box |
| 袋 | bag | 粉料/冻品/小物料 |
| 桶 | bucket | 液体大包装（含 drum） |
| 瓶 | bottle | 单瓶包装 |
| 卷 | roll | 包材类（保鲜膜/锡纸） |

### 3.3 设计逻辑  
- box 不区分 carton / outer_box → 避免概念混乱  
- bucket 合并 drum → 过度区分导致团队记不住  
- 保留 roll（包材特性强）  

---

# 四、使用单位 Usage Unit（厨房配方/出品单位）

### 4.1 定义说明  
Usage Unit 用于厨房，强调“操作习惯”与“出品标准”。

必须可被转换为 Base Unit。

---

### 4.2 使用单位（精简合并后）

| 中文 | 英文 | 说明 |
|------|--------|--------|
| 份 | serving | 出品单位 |
| 勺 | spoon | 大/小勺统一（重量由系统记录） |
| 杯 | cup | 冰杯/普通杯统一（容量由系统记录 ml） |
| 球 | scoop | 常用于固体球状 |

### 4.3 设计逻辑  
- 所有勺统一为 spoon → 防止 tsp/tbsp/spoon 混乱  
- 所有杯统一为 cup → 冰量/容量通过数字表示  
- scoop 保留 → 与甜品/火锅球类产品强相关  

---

# 五、价格体系 Price System（与单位完全对应）

价格体系同样分成三类，与单位结构一一对应：

```
包装价格 → 基础价格 → 使用价格
```

这确保成本计算可控、透明、可追溯。

---

## 5.1 包装价格 Package Price  
用于采购，来源于供应商。

```
price_per_case
price_per_box
price_per_bag
price_per_bucket
price_per_bottle
price_per_roll
```

---

## 5.2 基础价格 Base Price  
用于库存成本核算，是系统最重要的价格。

```
price_per_g
price_per_ml
price_per_piece
price_per_pack
```

---

## 5.3 使用价格 Usage Price  
用于配方（自动计算得出）。

```
price_per_serving
price_per_spoon
price_per_cup
price_per_scoop
```

系统根据基础价格自动换算：

```
price_per_spoon = spoon_g × price_per_g
price_per_cup = cup_ml × price_per_ml
price_per_scoop = scoop_g × price_per_g
price_per_serving = Σ(所有食材 usage_price)
```

# 七、一句话总结（最终原则）

> **最小单位用 Base Unit，采购用 Package Unit，厨房配方用 Usage Unit。  
所有价格必须严格使用 price_per_单位 的命名方式，实现自动化、标准化、可扩展。**
